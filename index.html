  • Focus on understanding AWS shared responsibility model — it appears in many questions

Quiz 1 — Sample Question
Cloud Concepts (24%)
An application experiences unpredictable traffic patterns with sudden spikes. The infrastructure automatically adds servers during spikes and removes them when traffic normalizes. What is this capability called?
  • Vertical scaling
  • ✓ Horizontal scaling with elasticity
  • Load balancing
  • Fault tolerance
Show Explanation

- Why correct: Horizontal scaling (adding/removing servers) combined with elasticity (automatic adjustment based on demand) describes the scenario where servers are automatically added during spikes and removed when traffic decreases. - Why A is wrong: Vertical scaling means increasing the size of existing servers, not adding/removing servers automatically. - Why C is wrong: Load balancing distributes traffic across servers but doesn't automatically add or remove servers based on demand. - Why D is wrong: Fault tolerance is about maintaining operations during failures, not about scaling based on traffic patterns.

Quiz 2 — Sample Question
Cloud Concepts (24%)
A team is designing a workload that must continue operating during component failures. They want to validate that their backup, failover, and restore steps work before an incident occurs. Which design principle from the Reliability pillar of the AWS Well-Architected Framework best matches this approach?
  • Adopt a consumption model
  • ✓ Test recovery procedures
  • Measure overall efficiency
  • Implement a strong identity foundation
Show Explanation

- Why correct: **Test recovery procedures** is a key design principle of the Reliability pillar. Practicing and validating backup, failover, and restore processes helps ensure the workload can recover as designed during real failures. - Why A is wrong: **Adopt a consumption model** is associated with Cost Optimization (pay only for what you use). - Why C is wrong: **Measure overall efficiency** is associated with Cost Optimization (understand outcomes versus spend). - Why D is wrong: **Implement a strong identity foundation** is a Security pillar design principle.

  • ✓ Amazon DynamoDB
  • Amazon VPC
  • Amazon EBS volumes

  • AWS Firewall Manager

  • ✓ AWS Shield

Show Explanation

Quiz 10 — Sample Question
Security and Compliance (30%)
A company stores sensitive customer records in an Amazon S3 bucket and requires that all objects use server-side encryption with AWS KMS keys (SSE-KMS). Under the shared responsibility model, who is responsible for configuring the bucket settings and policies to enforce this requirement?
  • AWS only
  • Neither; encryption is automatic
  • ✓ The customer only
  • Both AWS and the customer
Show Explanation

- Why correct: The customer is responsible for security IN the cloud, including configuring Amazon S3 bucket policies and encryption settings (such as enforcing SSE-KMS) to meet their compliance requirements. - Why AWS only is wrong: AWS provides and secures the underlying S3 service and encryption capabilities, but AWS does not configure customer-specific data protection requirements. - Why "Neither; encryption is automatic" is wrong: S3 can encrypt objects at rest by default, but enforcing a specific encryption requirement (like SSE-KMS) and validating compliance is the customer's responsibility. - Why both AWS and the customer is wrong: AWS is responsible for the security OF the cloud and provides encryption features, but the customer owns the configuration and use of encryption controls for their data.

  • Single Region deployment only
  • On-premises servers only
  • ✓ Multi-Region deployment with CloudFront
  • Single AZ deployment
Show Explanation

Deploying origins in multiple Regions improves resilience because the application can continue serving content if an origin Region has an outage and can reduce latency for cache misses by serving from a closer Region. - Why A is wrong: A single-Region deployment does not provide Regional disaster recovery and can increase latency for users far from the origin when content is not already cached. - Why B is wrong: On-premises servers do not provide a globally distributed edge network like CloudFront and would require the company to build and manage global distribution and failover. - Why D is wrong: A single Availability Zone does not provide Regional resilience and does not address global latency for worldwide viewers.

  • 99.9999999% (9 nines)

The distinction matters because S3 achieves 11 nines through redundant storage across multiple devices and multiple Availability Zones.

Quiz 19 — Sample Question
Cloud Technology and Services (34%)
A company runs a web application in a single AWS Region and needs it to remain available if one data center fails. What is the primary benefit of deploying across multiple Availability Zones?
  • ✓ High availability and fault tolerance
  • Automatic scaling of compute resources to match demand
  • Lower latency for end users in other geographic regions
  • Reduced costs by running fewer redundant resources
Show Explanation

- Why correct: Deploying across multiple Availability Zones improves high availability and fault tolerance by running the workload in physically separate locations within the same Region. If one AZ experiences a failure, the application continues running in another AZ. - Why Automatic scaling of compute resources to match demand is wrong: Automatic scaling is provided by AWS Auto Scaling, not by deploying across multiple AZs. Multi-AZ provides redundancy, while Auto Scaling adjusts capacity based on demand — they are complementary but distinct concepts. - Why Lower latency for end users in other geographic regions is wrong: Availability Zones are located within a single Region. To reduce latency for users in other geographic regions, you would deploy in multiple Regions or use a content delivery network like Amazon CloudFront with edge locations. - Why Reduced costs by running fewer redundant resources is wrong: Multi-AZ deployments actually increase costs because you are running duplicate resources in separate Availability Zones. The trade-off is higher cost for improved availability.

Quiz 20 — Sample Question
Billing, Pricing, and Support (12%)
A company has an upcoming event and must guarantee that it can launch specific EC2 instances in a particular Availability Zone when needed. What is an EC2 Capacity Reservation?
  • Free capacity for new customers
  • Unused EC2 capacity at discounted rates
  • ✓ Reserved capacity in a specific AZ that you can use anytime
  • A discount for long-term commitment
Show Explanation

- Why correct: A Capacity Reservation reserves compute capacity for a specific instance type and tenancy in a specific Availability Zone, ensuring that capacity is available when you need it. You pay for the reservation at the applicable On-Demand rate whether or not you use it. - Why 'Free capacity for new customers' is wrong: The AWS Free Tier offers limited free usage for eligible services, but it does not reserve EC2 capacity. - Why 'Unused EC2 capacity at discounted rates' is wrong: Spot Instances provide access to unused EC2 capacity at discounted rates, but capacity is not guaranteed and instances can be interrupted. - Why 'A discount for long-term commitment' is wrong: Reserved Instances and Savings Plans focus on discounted pricing for committed usage; a Capacity Reservation is specifically for guaranteeing capacity in an Availability Zone (and can be combined with discounts separately).

Show Explanation

Quiz 24 — Sample Question
All Domains (Comprehensive Review)
A company is reviewing its AWS security posture. Which THREE actions align with AWS security best practices? (Select THREE)
  • ✓ Enable AWS CloudTrail logging in all Regions
  • Share IAM user credentials among team members
  • ✓ Enable MFA on all privileged IAM accounts
  • ✓ Use IAM roles instead of long-term access keys for applications
  • Store access keys in application source code
  • Disable all CloudWatch alarms to reduce costs
Show Explanation

- Why A is correct: Enabling CloudTrail logging across all Regions ensures complete visibility into API activity for security auditing and incident investigation, even in Regions you do not actively use. - Why B is wrong: Sharing credentials violates IAM best practices. Each person should have their own IAM identity so actions can be traced to individuals for accountability and auditing. - Why C is correct: Multi-factor authentication (MFA) adds a second layer of protection beyond a password. It is especially critical for privileged accounts that can modify security settings or access sensitive data. - Why D is correct: IAM roles provide temporary, automatically rotated credentials, eliminating the risk of exposed or forgotten long-term access keys. AWS recommends roles for applications running on EC2 and other services. - Why E is wrong: Embedding access keys in source code is a critical security anti-pattern. Keys can be leaked through version control repositories. Use IAM roles or AWS Secrets Manager instead. - Why F is wrong: CloudWatch alarms are essential for detecting operational and security anomalies. Disabling them removes visibility into potential threats and service issues.
